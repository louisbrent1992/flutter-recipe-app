## App Store Review Notes

### Welcome Credits Feature

**New User Experience:**
- Create a new account (email, Google, or Apple Sign In)
- Upon successful signup, user automatically receives:
  - 5 Recipe Import Credits
  - 5 Recipe Generation Credits
- Credits are visible in the app's credit balance
- No credit card required for welcome credits
- This is a permanent feature, not a temporary promotion

**Implementation:**
- Welcome credits are granted once per user account
- Credits are stored in Firestore user document
- Credits never expire
- Cannot be revoked or transferred

### Free Trial Feature

**7-Day Unlimited Trial:**
- All subscription plans include a 7-day free trial
- During trial: **UNLIMITED imports and generations** (no credit limits)
- After trial: Revert to plan's monthly credit allocation
- Ad-free access is immediate (including during trial)
- No charge if cancelled during trial period

**Implementation:**
- Trial is marked with `trialActive: true` in Firestore
- `trialActive` flag grants unlimited usage for imports/generations
- Trial expires after 7 days (`trialEndAt` timestamp)
- After trial ends, credits are granted based on subscription plan
- One trial per user account (tracked via `trialUsed` flag)

### Privacy & Data Usage

**Data Collection:**
- User account information (email, name)
- Recipe preferences and dietary restrictions
- Purchase history and subscription status
- Credit balance and usage history
- User-created and imported recipes

**Third-Party Services:**
- Firebase Authentication (user accounts)
- Firebase Firestore (data storage)
- Recipe Generation API (smart recipe generation)
- RocketAPI (social media import)
- Google Ads (when not ad-free)

**Data Retention:**
- User data retained while account is active
- Deletion available through Settings ‚Üí "Delete Account"
- Purchase history retained per App Store requirements

### Server-Side Receipt Validation

All purchases are validated server-side:
1. Client receives purchase from App Store/Play Store
2. Receipt sent to RecipEase backend
3. Backend validates with Apple/Google
4. Credits/features granted only after validation
5. Purchase recorded in Firestore with receipt

This prevents:
- Fraudulent purchases
- Modified receipts
- Duplicate credit grants
- Unauthorized access

### Subscription Auto-Renewal

All subscriptions auto-renew according to Apple's guidelines:
- Users are clearly informed before purchase
- Trial terms displayed prominently
- Cancellation instructions in Settings
- Auto-renewal notice in subscription description
- Manage subscription link provided in app

### Restore Purchases

Users can restore purchases from Settings:
1. Navigate to Settings
2. Tap "Restore Purchases"
3. App queries App Store/Play Store
4. Non-consumables and active subscriptions restored
5. Consumable credits are NOT restored (as per store policies)
